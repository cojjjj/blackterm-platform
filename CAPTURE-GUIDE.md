# BLACKTERM Screenshot Capture Guide

## Required screenshots

Capture these eight screens:

1. Mission Control
2. Investigation Workspace
3. Threat Intelligence Center
4. Attack Surface Intelligence
5. Global Intelligence Map
6. Relationship Explorer
7. AI Investigation
8. Reporting

## Capture standard

- Use the same BLACKTERM theme for every image.
- Use a 16:9 application window where possible.
- Recommended capture size: 1600 × 900 or 1920 × 1080.
- Keep the full application window visible.
- Hide personal information, API keys, local paths, usernames, and private targets.
- Use a safe example target such as `example.com`, `example.org`, or reserved documentation IPs.
- Avoid empty screens. Populate each view with believable sample investigation data.
- Keep text readable at GitHub width.
- Close unrelated programs and notification overlays.
- Do not include the Windows taskbar unless it is intentionally part of the presentation.

## Safe sample data

Use documentation-only values such as:

- Domain: `example.com`
- IPv4: `192.0.2.24`, `198.51.100.17`, `203.0.113.42`
- Case: `CASE-2026-014`
- Analyst: `BLACKTERM OPERATOR`
- Organization: `EXAMPLE RESEARCH NETWORK`
- Status: `AUTHORIZED LAB`
- Threat result: `SUSPICIOUS — MODERATE CONFIDENCE`

The IP ranges above are reserved for examples and documentation.
