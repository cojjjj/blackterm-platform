# Security Policy

## Supported Version

The latest public preview is the supported development version.

## Reporting a Vulnerability

Do not open a public issue containing sensitive exploit details, credentials, private targets, or personal information.

Report security concerns privately to the repository owner through an appropriate private contact channel on their GitHub profile. Include:

- A clear description of the issue
- Affected file or component
- Reproduction steps
- Potential impact
- A suggested fix, when available

Please allow reasonable time for review before public disclosure.

## Scope

Security reports should concern vulnerabilities in BLACKTERM itself. Reports about third-party services found during authorized scans should be sent to the affected service owner, not posted in this repository.

## Responsible Use

BLACKTERM is intended for education, defensive security, lab environments, and explicitly authorized testing. Users are responsible for complying with applicable laws and obtaining permission before scanning systems.
